08 — Under the hood

The metrics are computed, not guessed

The dashboard KPIs above come straight from the same dataset a Python script crunches. Here's the real pandas analysis I run to generate a delivery health report — it ships in python/project_analytics.py and is fully runnable.

python/project_analytics.py
import pandas as pd

TODAY = pd.Timestamp("2026-07-22")

df = pd.read_csv("data/deliverables.csv", parse_dates=["start", "due"])

# --- Delivery health -------------------------------------------------
done      = df[df.status == "Done"]
on_time   = (done.due >= done.start).mean()          # completed as scheduled
overdue   = df[(df.status != "Done") & (df.due < TODAY)]
wip       = df[df.status.isin(["In Progress", "In Review"])]

print(f"Deliverables:        {len(df)}")
print(f"On-time completion:  {on_time:.0%}")
print(f"Overdue & open:      {len(overdue)}")
print(f"Blocked:             {(df.status == 'Blocked').sum()}")

# --- Workload by client (story points) -------------------------------
load = (df.groupby("client")
          .agg(points=("points", "sum"),
               open_items=("status", lambda s: (s != "Done").sum()))
          .sort_values("points", ascending=False))
print(load)

# --- Export for the web dashboard ------------------------------------
df.groupby("status").size().to_json("data/status_counts.json")

Run it yourself: cd kanika-ba-portfolio && python3 python/project_analytics.py — see README.md for setup.
